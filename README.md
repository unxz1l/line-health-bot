# LINE Health Bot — 阿臻

A LINE Bot that sends daily health reminders and replies to messages with warmth. Built for my family's elders, running on Cloudflare Workers (free tier).

## What It Does

**Scheduled push reminders** (via Cron Triggers):
- 08:00 TST — morning medication reminder
- 20:00 TST — evening medication reminder
- 22:00 TST — bedtime reminder

Each reminder is generated by an LLM so the wording varies day-to-day. If the LLM is unavailable, a random fallback template is used instead — the reminder always goes out.

**Chat replies** (via Webhook):
When a family member sends a message, the bot replies as "阿臻" — a caring, slightly playful younger family member. It can chat casually, encourage healthy habits, and provide emotional support. It strictly refuses to give medical advice. Emoji usage is limited to at most one per message, with varied selection to keep conversations natural.

## Architecture

```
LINE Platform
  │
  ├─ Webhook POST ──→ Cloudflare Worker (fetch handler)
  │                      ├─ Verify HMAC-SHA256 signature
  │                      ├─ Handle 「我的ID」 command
  │                      └─ Call Groq LLM → reply via LINE Reply API (free)
  │
  └─ Cron Triggers ──→ Cloudflare Worker (scheduled handler)
                         ├─ Generate reminder via Groq LLM (or fallback)
                         └─ Push to each user via LINE Push API (quota-limited)
```

```
src/
├── index.js      # Worker entry — fetch + scheduled handlers
├── webhook.js    # Webhook signature verification + message routing
├── line.js       # LINE API wrappers (reply, push, signature)
├── llm.js        # Groq API calls (chat reply + reminder generation)
├── cron.js       # Cron dispatch logic + multi-user push
├── fallback.js   # Fallback message templates
└── logger.js     # Structured JSON logging
```

## Prerequisites

- [Cloudflare account](https://dash.cloudflare.com/sign-up) (free plan works)
- [LINE Official Account](https://developers.line.biz/console/) with Messaging API enabled
- [Groq API key](https://console.groq.com/) (free tier)
- Node.js 18+

## Setup

1. **Clone and install**

   ```sh
   git clone <repo-url> && cd line-health-bot
   npm install
   ```

2. **Set secrets** (never commit these)

   ```sh
   npx wrangler secret put LINE_CHANNEL_SECRET
   npx wrangler secret put LINE_CHANNEL_ACCESS_TOKEN
   npx wrangler secret put GROQ_API_KEY
   ```

3. **Configure users**

   Edit `USER_IDS` in `wrangler.toml` — comma-separated LINE User IDs. See [Adding a Family Member](#adding-a-family-member) below.

4. **Deploy**

   ```sh
   npm run deploy
   ```

5. **Set webhook URL** in LINE Developer Console

   `https://line-health-bot.<your-subdomain>.workers.dev`

   Enable "Use webhook" and disable "Auto-reply messages".

## Environment Variables

| Variable | Type | Description |
|---|---|---|
| `LINE_CHANNEL_SECRET` | Secret | HMAC key for webhook signature verification |
| `LINE_CHANNEL_ACCESS_TOKEN` | Secret | LINE Messaging API bearer token |
| `GROQ_API_KEY` | Secret | Groq API key for LLM calls |
| `USER_IDS` | Var | Comma-separated LINE User IDs for push reminders |

## Free Tier Budget

LINE's free plan allows **200 push messages/month**. Reply messages are unlimited.

```
3 pushes/day × 30 days × N users = 90N messages/month

1 user  →  90/month ✅
2 users → 180/month ✅
3 users → 270/month ❌ exceeds limit
```

Cron reminders use push messages. Chat replies use reply messages (free, unlimited).

## Adding a Family Member

1. Have them add the bot as a LINE friend
2. They send the message: `我的ID`
3. The bot replies with their LINE User ID
4. Add that ID to `USER_IDS` in `wrangler.toml`, comma-separated
5. Redeploy: `npm run deploy`
